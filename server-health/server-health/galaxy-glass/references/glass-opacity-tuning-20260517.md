# 玻璃层不透明度调优记录 (2026-05-17)

## 问题

用户要求强模糊（毛玻璃）效果，初始实现用 `rgba(255,255,255,0.04)`（4% 白底）+ `backdrop-filter: blur(80px)`。但用户反馈"无模糊"。

## 原因

1. **壁纸太暗：** GalaxyGlass 背景壁纸经 `brightness(0.7)` 处理，luma 仅 ~23/255。4% 白底在几乎黑色的壁纸上视觉≈无。
2. **内容层全透明：** 文字区（content wrapper）没有背景色，看起来文字直接浮在壁纸上，玻璃效果不可见。
3. **Chrome clip-path bug：** `clip-path` + `backdrop-filter` 同元素 → Chrome 丢弃模糊。必须分两层。

## 最终调优值

### NodeCard（服务器卡片）
| 层 | 属性 | 值 |
|---|---|---|
| 玻璃层 | `background` | `rgba(255,255,255,0.15)` |
| 玻璃层 | `backdrop-filter` | `blur(80px) saturate(1.2)` |
| 内容层 | `background` | `rgba(255,255,255,0.02)` |
| 内容层 | `padding` | `20px` (`px-5 py-5`) |

### StatsBar（顶栏统计卡）
| 层 | 属性 | 值 |
|---|---|---|
| 玻璃层 | `background` | `rgba(255,255,255,0.18)` |
| 玻璃层 | `backdrop-filter` | `blur(60px) saturate(1.2)` |
| 内容层 | `background` | `rgba(255,255,255,0.02)` |
| 内容层 | `padding` | `14px 16px` (`px-4 py-3.5`) |

## 演进过程

| 迭代 | NodeCard 玻璃 | StatsBar 玻璃 | 内容层 | 用户反馈 |
|---|---|---|---|---|
| 1 | 0.04 (4%) | 0.06 (6%) | 透明 | "没有模糊" |
| 2 | 0.08 (8%) | 0.10 (10%) | 透明 | "无模糊，透明" |
| 3 ✅ | **0.15 (15%)** | **0.18 (18%)** | **0.02 (2%)** | 已确认 |

## 关键经验

- **暗色壁纸上玻璃层要 15%+ 才可见。** 低于 10% 在白底上可能够用，但在深色 OLED 背景上几乎看不出。
- **内容层必须有一点底色。** 全透明时文字区看到的是未经玻璃处理的壁纸，视觉上"没有毛玻璃"。
- **不要用数值推理代替视觉验证。** 技术上 backdrop-filter: blur(80px) 是"强模糊"，但视觉上取决于底层壁纸内容。
