# JS 语法解析失败根因分析（v2.2.0 踩坑）

## 问题

Node.js `--check` 报 `SyntaxError: Unexpected string`，指向详情页 tags 生成代码行。

## 根因

字符串拼接中使用了 `'"'` 和 `">"` 的混合：

```js
return'<span class="tag-chip'+tc2[i%4]+'">'+t.trim()+'</span>'
```

在单引号字符串中 `">"` 本应正确（`\"` 是合法转义），但某些环境下 Node.js 解析失败。
具体表现为 `--check` 始终报错，但从新文件写入相同的逻辑字符串却能通过。

疑似原因：Python 的 re.sub 或文件写入引入了肉眼不可见的字符差异，导致 Node 的词法分析器看到不同的字节序列。

## 修复

### 方法一：for 循环替代 map（推荐，最终采用）

```js
var th='';
if(tags.length){
  th='<div class="tags-list">';
  for(var _i=0; _i<tags.length; _i++){
    th+='<span class="tag-chip'+(tc2[_i%4]||'')+'">'+tags[_i].trim()+'</span>';
  }
  th+='</div>';
}
```

**优点**：完全避免 `return` 内的字符串拼接，逻辑更清晰。

### 方法二：分离内部函数（未采用）

```js
function tagChip(t,i){
  return'<span class="tag-chip'+(tc2[i%4]||'')+'">'+t.trim()+'</span>';
}
var th = tags.length?'<div class="tags-list">'+tags.map(tagChip).join('')+'</div>':'';
```

**注意**：`node --check` 同样报错，说明问题不在 inline function 本身。

## 防御

- 任何涉及 template 字符串拼接的复杂表达式，优先用 for 循环
- 修改后必须 `node --check` 验证
- 如果 --check 报错但肉眼看不出来，用 `node -e "new Function(code)"` 逐段验证
- 写到 `.js` 文件再跑 `node`（不是 `node --check`）可以获得更详细的错误信息
