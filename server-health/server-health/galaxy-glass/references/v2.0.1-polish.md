## 数据 URI + innerHTML 坑（v2.0.3 新增）

### 现象
详情页 meta 行显示 `" alt="" style="width:14px;..." loading="lazy" onerror="this.style.display='none'"> 🇺🇸 · kvm · Debian`

### 根因
`getOSIcon()` 返回 `data:image/svg+xml,<svg xmlns="..."...>`。通过 `innerHTML` 插入 `<img src="...data URI...">` 时，data URI 内的 `"` 字符提前截断了 `src` 属性。浏览器解析为：
```html
<img src="data:image/svg+xml,<svg xmlns="http://www.w3.org/2000/svg" ... ">
```
第一个 `"`（`xmlns="http://...`）被当作 `src` 的结束，后面所有内容变成文本节点。

### 修复
```js
// before
return 'data:image/svg+xml,<svg ...>...</svg>';
// after
var svg = '<svg ...>...</svg>';
return 'data:image/svg+xml,' + encodeURIComponent(svg);
```

### 规则
任何通过 innerHTML 插入的 `<img src="data:...">`，如果数据中包含 `<`、`>`、`"`，必须用 `encodeURIComponent()` 编码。
