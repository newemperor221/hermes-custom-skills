# 暗色毛玻璃设计规范（Research-Backed）

> 来源：uxpilot.ai glassmorphism UI 指南 + 本项目的实战验证（2025-2026）

## 核心原则

### 1. 不要到处用毛玻璃

毛玻璃应该只用在**关键的、高优先级**的元素上：
- 导航栏（如果用户要的话，但目前项目偏好透明）
- 主卡片／主面板
- 模态框

次要卡片、普通面板用纯色玻璃背景或无玻璃。每多一个 `backdrop-filter` 就多一份 GPU 开销。

### 2. blur 值范围

| 类型 | 推荐 blur | 说明 |
|------|-----------|------|
| 导航栏 | 4-10px | 导航栏 blur 不需要太大，关键在半透明底色 |
| 主卡片 | 4-12px | 毛玻璃卡片 blur 4-12px 视觉最佳 |
| 下拉菜单 | 8-15px | 菜单浮在上层，需要跟背景区隔 |
| 强毛玻璃 | 15-30px | 仅用于需要强区隔的特殊元素 |

**不要用 40px+ 的 blur** — 视觉效果"脏"、GPU 渲染成本高。

### 3. 暗色模式毛玻璃配色

- 玻璃底色：用微彩色调（navy/violet/teal），不要纯白透明
- rim 边（过渡边框）：用微彩色系代替纯白边（如藏青、紫罗兰、青色）
- 文字在毛玻璃上的可读性：加 10-30% 半透明覆盖层 + 1px text-shadow
- 对比度：普通文字 ≥ 4.5:1，大号文字 ≥ 3:1（WCAG）

GalaxyGlass 当前玻璃配色：
```css
--glass-subtle:   rgba(255,255,255,0.04);  /* 极淡 */
--glass-bg:       rgba(255,255,255,0.06);  /* 默认玻璃 */
--glass-raised:   rgba(255,255,255,0.08);  /* 悬浮/抬起 */
--glass-hover:    rgba(255,255,255,0.10);  /* hover */
--glass-strong:   rgba(255,255,255,0.14);  /* 强玻璃 */
--glass-border:   rgba(255,255,255,0.10);  /* 默认边框 */
```

### 4. 深色背景上 backdrop-filter 不可见的陷阱

当 `backdrop-filter` 作用于纯色深色背景时，视觉上几乎没有效果。**模糊透明层在纯色背景上=不可见。**

解决方案（如需使用毛玻璃）：
- 背景色跟页面底色有细微色差（微偏蓝 vs 纯深灰）
- 加 `saturate(150%) brightness(1.15)` 增强模糊效果
- 透明度用 40% 而不是 75%
- 或者放弃毛玻璃导航栏，用完全透明 + 底部细边（当前项目偏好）

### 5. Fallback 兼容

```css
@supports (backdrop-filter: blur(4px)) {
  .glass { backdrop-filter: blur(4px); }
}
@supports not (backdrop-filter: blur(4px)) {
  .glass { background: rgba(0,0,0,0.4); }
}
```

### 6. 性能注意

- 限制模糊元素数量（不要在整页背景或大量重叠元素上用）
- 毛玻璃 + 视频背景 = 高 GPU 消耗，移动端注意卡顿
- blur 值 > 20px 在低端设备上可能卡
- 不要给 backdrop-filter 做过渡动画

## GalaxyGlass 具体应用

| 位置 | 当前状态 | 是否符合规范 |
|------|---------|-------------|
| 导航栏 | 完全透明 + 底部细边 | ✅ 符合（用户的明确偏好） |
| 统计卡片 | `--glass-bg` + `blur(24px)` | ⚠️ blur 24px 偏大，但卡在深色视频背景上用较少 |
| 节点卡片 | `--glass-bg` + `blur(60px)` | ❌ 60px 过大，建议降到 12-20px |
| 筛选器 | `--glass-subtle` + `blur(24px)` | ⚠️ 24px 偏大，建议降到 8-12px |
| 下拉菜单 | `--glass-bg` + `blur(20px)` | ✅ 符合规范 |
| 详情卡片 | `--glass-bg` + `blur(24px)` | ⚠️ 建议降到 12-16px |
