# v0.54.0 OAuth & Onboarding Flow Reference

## Onboarding Wizard Steps

As of v0.54.0 (May 19, 2026), the first-launch flow changed to a multi-step wizard:

1. **Mode Picker** (BootCheckGate) — "Run Custom" vs "Simple"
2. **Inference (Text)** — "Default" (routes through core's local Ollama) vs "Configure" (bring your own API key)
3. **Voice** — Default or Configure (ElevenLabs/Whisper)
4. **Connections (OAuth)** — Finish Setup

## CLI Limitation

The `inference prompt` and `inference chat` CLI commands only check the local Ollama backend even when cloud providers are configured. They fail with "local ai is disabled" if Ollama is not running. Cloud inference works through the RPC layer when the desktop app sends requests — do NOT use the CLI to test cloud provider configuration.

## Inference Default Behavior

Choosing "Default" for Inference sets the core to use local Ollama with:
- `active_backend: "ollama"`
- `chat_model_id: "gemma3:1b-it-qat"`
- `provider: "ollama"`

On a headless VPS without Ollama, this silently fails — the app loads but no chat works. The user must either install Ollama or reconfigure with a cloud API key.

## Cloud Provider Configuration

The core may auto-detect a DeepSeekV4 provider entry:
```json
{
  "auth_style": "bearer",
  "endpoint": "https://api.deepseek.com",
  "id": "p_custom_e2b4h",
  "label": "DeepSeekV4",
  "slug": "custom"
}
```

To set an API key for it:
```bash
openhuman-core inference update_model_settings \
  --api_key "sk-your-deepseek-key" \
  --chat_provider "p_custom_e2b4h" \
  --primary_cloud "p_custom_e2b4h"
```

## Callback Page Structure

A successful GitHub OAuth callback from `api.tinyhumans.ai` returns an HTML page with:
- Title: "Opening OpenHuman"
- Auto-redirect after 1.2s to the deep link
- Deep link format: `openhuman://auth?token=<jwt>&key=auth`
- A "Copy Link" button
- The page has a `<script>` that sets `window.location.href` after timeout

An expired-state response returns:
```
openhuman://#auth/github/error?error=Invalid or expired OAuth state
```

**Important:** The browser address bar shows `https://api.tinyhumans.ai/auth/github/callback?code=...&state=...` — this is NOT the deep link. The deep link is only visible in the **page content** (the `<a>` tag or Copy Link button). If the user reads you the address bar URL, the state has already expired by the time you curl it.

## JWT Payload

Example decoded JWT from a fresh session:
```json
{
  "userId": "6a0c485608cd70fb5dd49bc4",
  "iat": 1779192651,
  "exp": 1786968651
}
```
- `iat` = time of login
- `exp` = 90 days later
